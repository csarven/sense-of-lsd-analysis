Abstract

By testing different datasets and sections, the ongoing hypothesis will be to see whether the strength of the correlations between the datasets have anything to do with the degree of their semantic similarity.

H₁ There may be an unknown relationship.

Workflow

Figure

1. I take the datasets that have an observation for the reference period 2013. Initially, there are 1406 datasets available that has at least one observation for the year 2013. The correlation is computed by first taking two of those datasets and joining them on the reference area, and using their measure values for the correlation calculation using kendall (assuming here that the relationships are non-linear). So, we get a table with header: refArea, obsValue.x, obsValue.y (the obValue.x is coming from one dataset, and obsValue.y from the other dataset). I do this for each dataset combination. For the output, I have a list for datasetX, datasetY, correlation, p-value, n. datasetX and datasetY are just the identifiers for the datasets. I filter out the rows that has n < 10, reason being that, it is probably not significant enough. I don't exactly know what is significant for sure. The population size i.e., the number of potential reference areas that can have an observation is 260. That's the number of reference area codes in their classification, but it is not necessarily the case that every dataset has anything for all of those reference areas. The possible number of reference areas per dataset is unknown. I'm not sure what to do about this. Any ideas? In any case, for now, I throw out n < 10 (that's the number after the join took place for the correlation). I deal with the rest later i.e., I can increase this if I need to.

2. Before doing the semantic similarity, I first take a unique list of the dataset identifiers from datasetX and datasetY so that what's to be checked for their similarity is for those datasets. At this point, the list of unique datasets is 402. The similarity is based on titles of the datasets. They are in short sentences e.g., "Mortality rate, infant (per 1,000 live births)". After some minor clean-up, it turns into "Mortality rate infant". World Bank happens to put the attribute value in the brackets in a lot of the cases. I figure that this is not that interesting for the similarity so I throw that out. It is also very possible to come across datasets that differ only by one word e.g., "male" or "female", or age "0-16" or "17-25" etc. The semantic similarity algorithm is based on a "latent semantic index". Essentially it puts each dataset title into a cluster. The number of clusters can be adjusted. Right now, it is using default 200. Hard to qualify what it "should be". I'll have to test. It gives back an output for datasetX, datasetY, similarity. The similarity value ranges from -1 to 1, similar to correlation values.

3. Lastly, I (currently, until I investigate further as you've suggested) filter out similarity values with 0 or 1 (after getting their absolute). And correlation values from the other table with 0 or 1 (after getting their absolute), as well as the p-values > 0.05. Thinking that the p-values above that value is not significant enough. The final scatter plot is created joining the table with correlations (datasetX, datasetY, correlation, pValue, n) with the table for similarities (datasetX, datasetY, similarity). Naturally, the join is on datasetX, datasetY, taking similarity and correlation values as the x and y values respectively. Correlation method is again based on kendall. Since I'm not doing any model fitting, I'm not interested in which gets to be x or y.

An LSD Sense workflow is as follows:

  1. Hypothesis
  2. Determine datasets and configurations
  3. Get metadata of datasets.
  4. Get each dataset's observations.
  5. Create correlations and other analysis for each dataset pair combination.
  6. Create dataset metadata subset for semantic similarity.
  7. Create semantic similarity for each dataset pair combination.
  8. Create correlation and other analysis.
  9. Interpret analysis and validate hypothesis.

With the exception of determining which datasets to inspect,and the system configuration, the LSD Sense implementation is automated.
